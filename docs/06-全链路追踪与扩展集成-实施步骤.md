### 额外优化：展开时高亮当前步骤

在 expander 内部添加简要摘要，让用户不展开也能快速了解每步做了什么：

```python
# 在 st.expander 的 label 中增加简要信息
def _step_summary(step: dict) -> str:
    """生成步骤的一行摘要"""
    if "调用工具" in step.get("action", ""):
        tool = step.get("tool", "未知")
        args = step.get("input", {})
        # 提取关键参数
        key_args = []
        for k, v in args.items():
            if isinstance(v, str) and len(v) < 30:
                key_args.append(f"{k}={v}")
        arg_str = ", ".join(key_args[:2]) if key_args else ""
        return f"🔧 {step['tool']}({arg_str})"
    elif "综合回复" in step.get("action", ""):
        thought = step.get("thought", "")
        return f"💬 {thought[:60]}..."
    elif "出错" in step.get("action", ""):
        return f"❌ {step.get('thought', '')[:60]}"
    return step.get("action", "")

# 然后在循环中使用：
for step in st.session_state.trace:
    emoji = step["action"].split(" ")[0] if " " in step["action"] else "📌"
    summary = _step_summary(step)
    with st.expander(
        f"{emoji} 第{step['num']}步 {summary}",
        expanded=False
    ):
        # ... 内部详情不变 ...
```

## 🔄 全链路数据流

```
用户输入 "帮我规划3天北京亲子游"
    │
    ▼
┌─ Agent (graph.py) ──────────────────────────────────────────┐
│  agent_node: LLM 决定调 search_attractions                   │
│    └→ token_tracker.record_call()   ← 步骤4 Token 追踪      │
│  tool_node: 执行工具                                          │
│    ├→ search_attractions  ──→ RAG (BM25 + BGE) ← 步骤3      │
│    ├→ get_weather          ──→ wttr.in API                  │
│    ├→ plan_route           ──→ AmapService    ← 步骤1       │
│    ├→ search_restaurants   ──→ AmapService    ← 步骤2       │
│    └→ search_hotels        ──→ AmapService    ← 步骤5       │
│  agent_node: LLM 综合结果 → 输出行程                          │
└──────────────────────────────────────────────────────────────┘
    │
    ▼ (Streamlit trace 捕获)
┌─ UI 渲染 (main.py) ────────────────────────────────────────┐
```
