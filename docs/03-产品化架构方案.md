## 三、安全体系（已完成）

| 组件 | 实现 | 文件 |
|------|------|------|
| JWT 认证 | access_token (15min) + refresh_token (7d) | [auth.py](../src/server/auth.py) |
| 密码哈希 | bcrypt（passlib fallback） | 同上 |
| API Key | B2B 客户独立 Key，可限定次数/有效期 | [models.py](../src/server/models.py) |
| 限流 | Token Bucket：免费 3次/天，Pro 100次/天；IP 30次/分钟 | [middleware.py](../src/server/middleware.py) |
| 熔断器 | LLM API 连续失败 5 次 → 熔断 30s → 半开探测 | 同上 |
| 请求追踪 | X-Request-ID 全链路透传 | 同上 |

## 四、数据模型

### PostgreSQL 核心表

| 表 | 关键字段 | 说明 |
|----|---------|------|
| `users` | id, email, nickname, hashed_password, role, created_at | 用户账户 |
| `trips` | id, user_id, city, days, itinerary_json (JSONB), status, share_token | 行程数据 |
| `tool_call_logs` | id, user_id, session_id, tool_name, input_json, output_json, latency_ms, token_count | 审计日志 |
| `api_keys` | id, user_id, key_hash, rate_limit, expires_at | B2B API Key |
| `user_preferences` | id, user_id, budget_level, travel_style, dietary | 用户偏好 |

### Redis 缓存策略

| 用途 | TTL | Key 模式 |
|------|-----|---------|
| 工具调用缓存 | 5 min | `tool:{name}:{hash(args)}` |
| 会话状态 | 30 min | `session:{session_id}` |
| 限流计数 | 随窗口 | `rate:{user_id}:{window}` |

## 五、成本控制

| 策略 | 节省比例 | 实现方式 | 状态 |
|------|---------|---------|------|
| System Prompt 缓存 | ~40% | LangChain cache 模块 | ⏳ |
| 工具调用去重 | ~20% | Redis `hash(tool+args)` → 直接返回 | ✅ |
| Token 预算熔断 | — | 单次请求 > 20000 tokens 自动截断 | ✅ |
| Token 追踪 | — | 每次 LLM 调用记录 cost | ✅ |

**成本目标**：单次行程规划 ¥0.03 → ¥0.008（降幅 73%）

## 六、商业模式

### 三条路径

| 路径 | 模式 | 目标客户 | 收入模型 |
|------|------|---------|---------|
| **B2B SaaS** | 旅行社智能规划工作台 | 中小旅行社 | ¥3000/月/席位 |
| **B2C 交易佣金** | 酒店 3-5% / 门票 5-8% | C 端用户 | 按交易抽成 |
| **B2C 订阅制** | Pro ¥29/月，家庭 ¥49/月 | 高频旅行用户 | 订阅费 |

> **推荐顺序**：B2B 先拿现金流 → C 端佣金规模化 → 订阅制锁高价值用户
