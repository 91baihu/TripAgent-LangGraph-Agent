## 一、项目定位

**TripAgent** 是一个 AI 驱动的旅行规划 Agent。用户用自然语言描述旅行需求，Agent 自主调用多个工具（景点搜索、天气查询、路线规划、餐厅推荐、酒店预订），通过 LangGraph ReAct 循环多步推理，输出完整的旅行行程方案。

**核心叙事线**：`需求 → 拆解 → 调工具 → 判断 → 再调 → 综合输出`

### Agent ReAct 循环

```
User input → [agent node: LLM decides] → conditional edge → [tools node: execute] → back to agent → ... → END
```

Agent 可自主调用的 **5 个工具**：

| 工具 | 功能 | 数据来源 |
|------|------|---------|
| `search_attractions` | 景点搜索（含 RAG 混合检索） | BM25 + BGE 向量 / Mock fallback |
| `get_weather` | 实时天气查询 | wttr.in API（可升级和风+彩云） |
| `plan_route` | 景点间距离/交通推荐 | 高德地图 API / Haversine fallback |
| `search_restaurants` | 周边美食搜索 | 高德 POI / Mock fallback |
| `search_hotels` | 酒店预订搜索 | 高德 POI / Mock fallback |

## 三、技术栈

| 层 | 技术 |
|---|------|
| Agent 框架 | LangGraph (`StateGraph`), LangChain tools (`@tool`) |
| LLM | DeepSeek Chat API (`langchain-deepseek`) |
| API 服务 | FastAPI (async), SSE 流式 (`sse-starlette`) |
| 数据库 | PostgreSQL 16 + SQLAlchemy 2.0 async (`asyncpg`) |
| 缓存 | Redis 7 (带 in-memory dict fallback) |
| 认证 | JWT (python-jose) + bcrypt |
| 原型 UI | Streamlit (chat + 推理追踪侧边栏) |
| Web 前端 | React 18 + TypeScript + Vite + Tailwind CSS |
| RAG 检索 | HybridRetriever (BM25 + BGE 向量混合) |
| 基础设施 | Docker Compose (api, postgres, redis, nginx, admin) |
| CI/CD | GitHub Actions (lint → test → build → deploy) |

## 七、面试叙事参考

### Agent vs RAG 区别
> 景小游是 RAG — 用户问「故宫几点关门」，系统检索知识库回答。TripAgent 是 Agent — 用户说「帮我规划3天北京游」，Agent 自己决定先搜景点、再查天气、算路线、排顺序、最后综合输出。RAG 是检索+生成，Agent 是推理+工具调用+生成。实际业务里 Agent 通常内置 RAG 作为其中一个工具。

### 为什么用 LangGraph
> 直接调 API 只能单轮问答。LangGraph 把 Agent 思考过程建模成状态图：agent 节点思考、tools 节点执行、条件边判断「信息够了就结束，不够继续调工具」。这个 ReAct 循环由 LangGraph 管理，不需要手写 while 循环。

### 从 Demo 到产品的关键改造
> Streamlit 原型验证了 Agent 可行性后，做了四件事让它可上线：① FastAPI 后端化 + SSE 流式 ② PostgreSQL 持久化 + Redis 缓存 ③ JWT 认证 + 限流 + 熔断器 ④ React 前端 + Docker Compose 一键部署。
