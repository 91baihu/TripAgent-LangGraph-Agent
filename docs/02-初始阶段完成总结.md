## 一、项目概述

TripAgent 是一个 AI 旅行规划 Agent，用户用自然语言描述旅行需求，Agent 自主调用多个工具、多步推理后输出完整旅行方案。核心叙事线为「需求 → 拆解 → 调工具 → 判断 → 再调 → 输出」的 ReAct 闭环。

## 二、已完成模块清单

### 2.1 Agent 编排引擎（`src/agent/`）

| 文件 | 完成内容 |
|------|---------|
| [state.py](src/agent/state.py) | `AgentState` 状态定义：`messages`（对话历史）、`next_step`（路由）、`travel_plan`（最终方案） |
| [prompts.py](src/agent/prompts.py) | 分层 System Prompt：角色定义层 → 工作流程层 → 输出格式约束层 → 注意事项层 |
| [graph.py](src/agent/graph.py) | ⭐ **LangGraph StateGraph 核心**：`agent` 节点（LLM 思考）→ 条件边（判断是否调工具）→ `tools` 节点（执行工具）→ 回到 agent。ReAct 循环完全由状态图驱动，非硬编码 while 循环。支持流式输出 (`streaming=True`) 和 LLM 调用异常处理 |

### 2.2 四个 LangChain Tool（`src/tools/`）

| 工具 | 文件 | 功能 | 数据来源 |
|------|------|------|---------|
| `search_attractions` | [attractions.py](src/tools/attractions.py) | 按城市+关键词搜索景点，支持「亲子」等特殊关键词过滤 | Mock 数据（北京 5 个 + 杭州 3 个） |
| `get_weather` | [weather.py](src/tools/weather.py) | 查询指定城市天气 | [wttr.in](https://wttr.in) 免费 API，含超时降级处理 |
| `plan_route` | [route.py](src/tools/route.py) | 计算两景点直线距离 + 推荐交通方式（步行/地铁/打车） | 12 个景点坐标，Haversine 近似算法 |
| `search_restaurants` | [restaurants.py](src/tools/restaurants.py) | 按城市+附近景点+预算等级搜索美食 | Mock 数据（北京 6 家 + 杭州 5 家），支持三级预算筛选 |

**设计要点**：每个工具使用 `@tool` 装饰器 + Google 风格 docstring，LangChain 自动解析参数类型和描述，生成 LLM 可理解的 function schema——这是 Agent 能「自主决定调用哪个工具」的基础。

### 2.3 RAG 混合检索模块（`src/rag/`）

| 文件 | 完成内容 |
|------|---------|
| [retriever.py](src/rag/retriever.py) | `HybridRetriever` 类，BM25（关键词）+ 向量检索框架，支持外部 JSON 数据文件加载和内置数据兜底 |
| [data/attractions.json](src/rag/data/attractions.json) | 8 个景区的结构化知识库（标题、城市、类型、时长、票价、亲子友好、详细描述） |

> **设计意图**：此模块预留了景小游 RAG 的接入接口。当前 `search_attractions` 使用 Mock 数据保证零依赖可运行，后续只需切换数据源即可升级为真正的混合检索。

### 2.4 Streamlit 交互界面（`src/main.py`）

| 功能 | 说明 |
|------|------|
| **左右双栏布局** | 左侧对话区 + 右侧推理追踪面板 |
| **推理过程可视化** | 实时捕获 `stream_mode="values"` 事件，展示每一步：调了什么工具 → 传了什么参数 → 返回了什么结果 → 最终如何综合 |
| **多轮对话记忆** | 每轮传入完整消息历史，支持「把第二天改成户外主题」这类增量修改 |
| **流式输出** | LLM 开启 `streaming=True` |
| **错误友好提示** | LLM 调用失败 / API Key 未配置时给出明确排查指引 |

## 七、启动验证步骤

```bash
# 1. 安装依赖（已安装可跳过）
pip install -r requirements.txt

# 2. 配置 API Key
# 编辑 .env，填入: DEEPSEEK_API_KEY=sk-your-key

# 3. 运行单元测试（不需要 API Key）
pytest tests/test_tools.py -v -k "not AgentIntegration"

# 4. 启动 Streamlit 前端
cd src && streamlit run main.py

# 5. 在浏览器中测试输入：
#    "帮我推荐北京好玩的"
#    "帮我规划3天北京亲子游，中等预算"
```
