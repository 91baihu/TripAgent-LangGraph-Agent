# 05 — 天气 API 升级方案

> **类型**：待实施方案 | **日期**：2026-07-12 | **状态**：⏳ 远期
> **关联**：[04-前端设计系统方案](04-前端设计系统方案.md) — 天气卡片的前端组件已就位，后端数据升级后可发挥完整能力

---

## 一、当前问题

```
现在: get_weather → wttr.in（免费无 Key） → 返回纯文本 "Clear 32°C 45% 15km/h"
                                                    ↓
                       前端 parseWeatherFromText() 用正则硬猜 → 极易解析失败
```

**瓶颈链**：
1. wttr.in 不稳定（无 SLA、无商业保障）
2. 返回纯文本 → 前端正则解析脆弱
3. 只有当天天气 → 无法做多日预报 → UI 只能展示单日
4. 中文城市名映射表手动维护（35 个城市）

---

## 二、API 选型：和风天气（主）+ 彩云天气（备）

| 维度 | 和风天气 (QWeather) | 彩云天气 (Caiyun) |
|------|-------------------|-------------------|
| **免费额度** | 1000次/天 | 1000次/天 |
| **7天预报** | ✅ 逐日 + 逐小时 | ✅ 逐小时（48h）+ 逐日（15天） |
| **实时天气** | ✅ 温度/体感/湿度/风力/能见度 | ✅ 同 |
| **分钟级降水** | ✅ 未来2小时 | ✅ 未来2小时 |
| **AQI 空气** | ✅ 6项指标 | ✅ 6项指标 |
| **生活指数** | ✅ 15项（穿衣/防晒/洗车...） | ✅ 8项 |
| **中文文档** | ✅ 完善 | ✅ 完善 |
| **注册** | 手机号即可，即时开通 | 手机号即可，即时开通 |

**结论**：两者都是国产，对中文城市支持天然好。推荐同时接入，互备 fallback。

---

## 三、目标架构

```
                    get_weather(city, date)
                           │
                    ┌──────┴──────┐
                    ▼             ▼
              和风天气 API    彩云天气 API
              (主, 1st)       (备, fallback)
                    │             │
                    └──────┬──────┘
                           ▼
                      Redis 缓存
                   (key=city+date, TTL=10min)
                           │
                    ┌──────┴──────┐
                    ▼             ▼
              结构化 JSON    降级 wttr.in
              (前端精确解析)   (最终兜底)
```

---

## 四、结构化输出

后端返回 **Markdown (人类可读) + Embedded JSON (前端可解析)**：

```markdown
## 北京 天气 · 晴

| 指标 | 数值 |
|------|------|
| 🌡️ 当前温度 | **32°C**（体感 35°C） |
| 💧 湿度 | 45% |
| 🌬️ 风力 | 东南风 3级 |
| 🫁 空气质量 | AQI 42 · 优 |
...

<!-- WEATHER_JSON
{
  "city": "北京",
  "temp": 32,
  "feels_like": 35,
  "condition": "晴",
  "condition_code": 100,
  "humidity": 45,
  "wind_scale": 3,
  "aqi": 42,
  "daily": [
    {"date": "2026-07-13", "temp_high": 33, "temp_low": 24, "condition_day": "多云"},
    ...
  ],
  "life_index": {"穿衣": "薄外套", "防晒": "强"},
  "source": "hefeng"
}
-->
```

前端优先解析 `<!-- WEATHER_JSON ... -->`，fallback 到正则解析 Markdown。

## 五、配置方式

```bash
# config/.env 中新增
HEFENG_API_KEY=your_key_here        # 和风天气 DevAPI Key
CAIYUN_API_KEY=your_key_here        # 彩云天气 API Key
```

注册地址：
- 和风天气：https://dev.qweather.com/
- 彩云天气：https://dashboard.caiyunapp.com/

## 六、改造对比

| 维度 | 现状 (wttr.in) | 改造后 (和风+彩云) |
|------|---------------|-------------------|
| 数据稳定性 | 无 SLA | 商业 API，99.9% 可用 |
| 数据丰富度 | 仅当前温度+天气 | 实时+逐时+7日预报+AQI+生活指数 |
| 前端解析 | 正则硬猜，容易失败 | JSON 精确解析，零歧义 |
| 中文城市 | 手动维护 35 个映射 | 原生支持，全球覆盖 |
| 成本 | 免费 | 免费（1000次/天 ≈ 月 3 万次） |
| 对 UI 的支撑 | 只能做单日卡片 | 可做 7 日预报条 + 空气质量环 + 生活建议 |
